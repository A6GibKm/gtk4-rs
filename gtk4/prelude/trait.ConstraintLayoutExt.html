fn add_constraint<P: IsA<Constraint>>(&self, constraint: &P)

Adds a Constraint to the layout manager.

The Constraint:source and Constraint:target properties of constraint can be:

  • set to None to indicate that the constraint refers to the widget using self
  • set to the Widget using self
  • set to a child of the Widget using self
  • set to a guide that is part of self

The self acquires the ownership of constraint after calling this function.

constraint

a Constraint

fn add_guide<P: IsA<ConstraintGuide>>(&self, guide: &P)

Adds a guide to self. A guide can be used as the source or target of constraints, like a widget, but it is not visible.

The self acquires the ownership of guide after calling this function.

guide

a ConstraintGuide object

fn observe_constraints(&self) -> Option<ListModel>

Returns a gio::ListModel to track the constraints that are part of self.

Calling this function will enable extra internal bookkeeping to track constraints and emit signals on the returned listmodel. It may slow down operations a lot.

Applications should try hard to avoid calling this function because of the slowdowns.

Returns

a gio::ListModel tracking self's constraints

fn observe_guides(&self) -> Option<ListModel>

Returns a gio::ListModel to track the guides that are part of self.
